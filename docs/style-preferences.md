# Style Preferences

## Product Direction

The initial site is a utilitarian CMS interface. It should prioritize clarity, speed, and reliable management workflows over marketing presentation.

## Visual Style

- Use a quiet, work-focused interface.
- Prefer high contrast and readable type.
- Avoid decorative backgrounds, large hero sections, and excessive animation.
- Keep page layouts predictable.
- Use simple tables and forms for administration screens.
- Use compact spacing where data density matters.

## UI Patterns

- Primary actions should be visually distinct.
- Destructive actions require confirmation.
- Disabled controls should explain the unavailable state where practical.
- Form errors should appear near the field they apply to.
- Success messages should be brief and dismissible when the UI grows.

## Content Style

- Use direct labels:
  - "Email"
  - "Password"
  - "Role"
  - "Status"
  - "Save"
  - "Revoke"
- Avoid vague labels such as "Submit" when a more specific verb is available.
- Use sentence case for headings and labels.

## Accessibility

- Every form input needs a label.
- Interactive controls need visible focus states.
- Color must not be the only signal for status or errors.
- Pages should remain usable with keyboard navigation.
